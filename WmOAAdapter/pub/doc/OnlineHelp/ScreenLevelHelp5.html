Configure Connection Type Screen

Use the Configure Connection Type screen to create a new connection and specify information that the Integration Server uses to connect and log into an Oracle Applications server.

  1. In the Oracle Apps Adapter section, provide values for the following fields.

    2. Parameter
    Description/Action

    Package

    The package in which to configure the connection.

    You must create the package using the Developer before you can specify it for this parameter. For general information about creating packages, see the webMethods Developer User’s Guide.

    Note: Configure the connection in a user-defined package rather than in the adapter’s package. See the webMethods Oracle Applications Adapter User’s Guide for other important considerations when creating packages for use with the Oracle Applications Adapter.

    Folder Name

    The folder in which to configure the connection.

    Connection Name

    The name you want to give the connection. Connection names cannot have spaces or use special characters reserved by the Integration Server or Developer. See the webMethods Developer User’s Guide for more information about the use of special characters in package, folder, and element names.

  2. In the Connection Properties section, provide values for the following fields:

    3. Parameter
    Description

    Oracle Apps Release version

    The version of the Oracle Applications server to which you are connecting.

    Database Server Host

    The host name of the Oracle Applications database to which you are connecting.

    Database Port Number

    The port number of the Oracle Applications database.

    Database Name

    The name of the Oracle Applications database.

    Database User Name

    The user name that the webMethods Integration Server must user to log into the Oracle Applications database. Typically the user name is APPS.

    The user name you specify must have access to the Oracle package that contains the Oracle Applications tables and the adapter database setup in the Oracle DBMS.

    Database Password

    The password that the webMethods Integration Server must user to log into the Oracle Applications database.

    Retype Database Password

    Confirms the password you specified in Database Password.

    Local/XA Transaction Control

    The type of transaction support that the connection provides. Select the following transaction type:

    LOCAL_TRANSACTION: The connection does not automatically commit transactions. You can manually define the transactions, or the Integration Server’s transaction manager will manage it for you. For more information about built-in transaction management services and instructions on managing transactions manually, see the webMethods Oracle Applications Adapter User’s Guide.

    DataSource Class Name

    The name of the JDBC driver’s DataSource class name. Only the Oracle Thin JDBC Driver is supported:

    oracle.jdbc.pool.OracleDataSource

    Network protocol

    A standard Oracle Applications DataSource property to indicate the name of the network protocol that the connection will use when connecting to the database.

    Leave this parameter blank.

    Other properties

    Enables you to provide additional JDBC driver DataSource properties for the Oracle Thin JDBC Driver. The following property is supported:

    driverType=thin

  3. In the Connection Management Properties section, provide values for the following fields:

    4. Parameter
    Description

    Enable Connection Pooling

    Enables the adapter to use connection pooling. Default: true.

    See the webMethods Oracle Applications Adapter User’s Guide for more information about connection pooling.

    Minimum Pool Size

    If connection pooling is enabled, this field specifies the number of connections to create when the connection is enabled. The adapter will keep open the number of connections you configure here regardless of whether these connections become idle. Default: 1.

    Maximum Pool Size

    If connection pooling is enabled, this field specifies the maximum number of connections that can exist at one time in the connection pool. Default: 10.

    Pool Increment Size

    If connection pooling is enabled, this field specifies the number of connections by which the pool will be incremented if connections are needed, up to the maximum pool size. Default: 1.

    Block Timeout (msec)

    If connection pooling is enabled, this field specifies the number of milliseconds that the Integration Server will wait to obtain a connection with the database before it times out and returns an error.

    For example, you have a pool with Maximum Pool Size of 20. If you receive 30 simultaneous requests for a connection, 10 requests will be waiting for a connection from the pool. If you set the Block Timeout to 5000, the 10 requests will wait for a connection for 5 seconds before they time out and return an error. If the services using the connections require 10 seconds to complete and return connections to the pool, the pending requests will fail and return an error message stating that no connections are available.

    If you set the Block Timeout value too high, you may encounter problems during error conditions. If a request contains errors that delay the response, other requests will not be sent. This setting should be tuned in conjunction with the Maximum Pool Size to accommodate such bursts in processing.

    Default: 1000.

    Expire Timeout (msec)

    If connection pooling is enabled, this field specifies the number of milliseconds that an inactive connection can remain in the pool before it is closed and removed from the pool.

    The connection pool will remove inactive connections until the number of connections in the pool is equal to the Minimum Pool Size. The inactivity timer for a connection is reset when the connection is used by the adapter.

    If you set the Expire Timeout value too high, you may have a number of unused inactive connections in the pool. This consumes local memory and a connection on your backend resource. This could have an adverse effect if your resource has a limited number of connections.

    If you set the Expire Timeout value too low, performance could degrade because of the increased activity of creating and closing connections. This setting should be tuned in conjunction with the Minimum Pool Size to avoid excessive opening/closing of connections during normal processing.

    Default: 1000.

    Startup Retry Count

    (For Integration Server 6.1 only.) The number of times that the system should attempt to initialize the connection pool at startup if the initial attempt fails. Default: 0.

    Startup Backoff Timeout (sec)

    (For Integration Server 6.1 only.) The number of seconds that the system should wait between attempts to initialize the connection pool.

  4. Click Save Connection.

    5. The connection you configured appears on the adapter’s Connections screen and in the Developer’s Navigation Panel.

    By default, when you configure a connection, it is not enabled. Use the Connections Screen to enable the connection.

Edit Connection Screen

Use the Edit Connection screen to update the properties for an existing connection. You cannot edit a connection if it is enabled. Use the Connections Screen to disable the connection.

Update the connection’s parameters by typing or selecting the values you want to specify.For descriptions of the connection’s parameters, see Configure Connection Type Screen.

When you are done making changes to the connection properties, click Save Changes.

Enable the connection when you are ready to use it.

To return to the Connection Types screen, click Return to Oracle Apps Adapter Connections.

View Connection Screen

Use the View Connection screen to view the properties for an existing connection.

For descriptions of the connection’s parameters, see Configure Connection Type Screen.

To return to the Connection Types screen, click Return to Oracle Apps Adapter Connections.

Copy Connection Screen

Use the Copy Connection screen to create a new connection by copying the properties for an existing connection.

Name the new connection, specify a Package Name and Folder Name, and edit any connection parameters as needed by typing or selecting the values you want to specify. For descriptions of the connection’s parameters, see Configure Connection Type Screen.

 

Note: You must enter and then retype a password before you can save the new connection.

Click Save Connection.

To return to the Connection Types screen, click Return to Oracle Apps Adapter Connections.
