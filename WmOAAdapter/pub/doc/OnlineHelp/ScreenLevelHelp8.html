Transaction Management Screens

Use the Transaction Mgmt screen to select one of the following options:

  • Transaction Configuration - Click to display the Transaction Configuration Page

    • Additional Transaction Configuration pages:

    New Transaction Page

    Edit transaction Transaction Page

    Error Service SQL for transaction Page

    Acknowledgement Service SQL for transaction Page

  • Transaction Import - Click to display the Oracle Applications Transaction Import Page
  • Transaction Export - Click to display the Oracle Applications Transaction Export Page

Transaction Configuration Page

Use the Transaction Configuration page to add, edit, or delete transaction definitions.

  1. From the Oracle Apps Release list, select the release of the Oracle Applications for associated with the transaction definition.
  2. In the Transaction Type field, select the type of transaction:

    3. IS-to-Oracle refers to transaction definitions that insert data from the webMethods Integration Server into Oracle Applications tables

    Oracle-to-IS refers to transaction definitions that select data from Oracle Applications and deliver it to the webMethods Integration Server

  3. The Current transaction_type Transactions list shows all current transaction definitions for the selected transaction type.
  4. Click one of the buttons:

    5. Button
    Description

    Add

    Displays the New Transaction Page which you use to create a new transaction definition.

    Edit

    Displays the Edit transaction Transaction Page which you use to edit a selected transaction definition from the Current transaction_type Transactions list.

    Delete

    Use to delete a selected transaction definition from the Current transaction_type Transactions list.

    Cancel

    Cancels the operation and returns you to the Transaction Mgmt screen.

New Transaction Page

Use the New Transaction Page to create a new transaction definition. The fields on this page correspond to the transaction type you selected on the Transaction Configuration page.

  1. Enter the name of the new transaction definition in the Transaction Name field. Click one of the following buttons:

    2. Button
    Description

    Submit

    Submits the name of the new transaction definition, and displays the Edit transaction Transaction Page page where you specify the remaining information.

    Reset

    Clears the Transaction Name field.

    Cancel

    Cancels the operation and returns you to the Transaction Configuration page.

Edit transaction Transaction Page

Use the Edit transaction Transaction page to specify the required information for a new transaction definition or to edit information for an existing transaction definition. The fields on this page correspond to the type of transaction definition you are working with.

For IS-to-Oracle transaction definitions:

  1. Enter the name of an open interface table in the Table Name field. If you want to specify that this table is required, check the Required option. Then click Submit.

    2. Repeat this step for each table you want the transaction to have access to. If you want to add a new table but a table is currently selected, click Cancel to de-select the current table.

    Each table added appears in the Current Tables area. If you want to delete a table from the list, select it and then click Delete Table.

  2. To specify SQL to check for errors that occurred during the import process, click Edit Error SQL. The adapter displays the Error Service SQL for transaction Page.

For Oracle-to-IS transaction definitions:

  1. Specify the top-level alias for the transaction, as follows:

    2. Field
    Description

    Alias Name

    The alias name.

    SQL

    Enter or edit the SQL statement that performs the query, or enter the name of a stored procedure to execute:

    To expose an SQL input parameter as a service input in the Developer, enclose the input parameter within an ampersand (&) and an asterisk (*). For example, &varname*. Stored procedure input parameters are exposed as service inputs in the Developer.

    To call a stored procedure, enter the stored procedure name in the format schema.package.procname. If the stored procedure is not located in a package, you do not have to enter the schema and package names.

  2. Click Submit.
  3. To specify a child for an alias, select the alias to which you want to add a child and click Next Level. If you want to add a new sibling alias but an alias is currently selected, click Cancel to de-select the current alias. Then specify the alias information, as follows:

    4. Field
    Description

    Alias Name

    The alias name.

    SQL

    Enter or edit the SQL statement that performs the query.

    Relations

    The relationship of the current alias to its parent alias.

    If you are creating an alias with a SELECT statement, Relations defines the WHERE clause and join parameters. Enter the values as follows:

    COLUMN = PARENT_ALIAS.COLUMN AND COLUMN2 = PARENT_ALIAS.COLUMN2 …

  4. Click Submit.
  5. To add SQL to acknowledge the transaction, click Edit Ack SQL. The adapter displays the Acknowledgement Service SQL for transaction Page.

Error Service SQL for transaction Page

Use the Error Service SQL for transaction page to specify the error SQL statements for an IS-to-Oracle transaction definition.

Add the SQL in the Error SQL field. To expose an SQL input parameter as a service input in the Developer, enclose the input parameter within an ampersand (&) and an asterisk (*). For example, &varname*. Stored procedure input parameters are exposed as service inputs in the Developer.

Then click one of the following buttons:

Button
Description

Submit

Submits the Error SQL for the transaction definition.

Reset

Clears the Error SQL field.

Cancel

Cancels the operation and returns you to the Transaction Configuration page.

Acknowledgement Service SQL for transaction Page

Use the Acknowledgement Service SQL for transaction page to add the SQL statements for the acknowledgement for an Oracle-to-IS transaction definition. The purpose of an Acknowledgement SQL is to prevent the query from retrieving the same piece of data again.

Add the SQL in the Acknowledgement SQL field, and then click one of the following buttons:

Button
Description

Submit

Submits the Acknowledgement SQL for the transaction definition.

Reset

Clears the Acknowledgement SQL field.

Cancel

Cancels the operation and returns you to the Transaction Configuration page.

Oracle Applications Transaction Import Page

Use the Oracle Applications Transaction Import page to import one transaction definition file at a time into the webMethods Repository. Transaction definition files must use the .txp file extension.

Files you want to import must be located in the following directory:

Integration Server_Directory\packages\WmOAAdapter\exchange

  1. Select a file from the Import Files list.

    2. If no files are listed, ensure that the transaction file to import is in the \exchange directory.

  2. From the Import to Oracle Applications Release list, select the release of Oracle Applications for which you are importing transaction definitions. The list defaults to the release from which the transaction definitions were exported.
  3. Select the transactions you want to import, as follows:

    4. Field
    Description

    Import this transaction

    Whether you want to import the transaction.

    Click Set to select all transactions to import. Click Clear to de-select all transactions selected for import.

    Import Name

    The name you want the transaction to have in your system. The default is the name as it is specified in the import file.

    Overwrite Existing

    Whether you want to overwrite transactions that are named the same as the import name.

    To overwrite all transactions, click Set. To de-select all transactions, click Clear.

  4. Click Import. The adapter displays the Oracle Applications Transaction Import Results screen, which lists the status of the transaction definition you imported.

Oracle Applications Transaction Export Page

Use the Oracle Applications Transaction Export page to export one transaction definition file at a time from the webMethods Repository. Transaction definition files must use the .txp file extension. Exported files are saved to the Integration Server_Directory\packages\WmOAAdapter\exchange\export directory.

  1. Select the release of the Oracle Applications transactions you want to export from the Oracle Apps Release list.
  2. Provide information about the export file, as follows:
    Field
    Description

    Oracle Apps Release

    The release of Oracle Applications from which you want to export transaction definitions.

    Export File

    The name of the export file. If the file does not currently exist, the adapter will create it for you.

    You do not have to specify the .txp file extension; it is supplied for you automatically.

    Overwrite Existing File

    Whether to overwrite an existing file if there is already a file with the name you specified in Export File.

    Create Host

    The machine on which the Integration Server is running.

    If you want to change the name of the host for security purposes, type a new name in the field.

    Description

    A description to identify the contents of the file.

  3. Select the transactions you want to export. To select all transactions to export, click Set. To de-select all transactions selected for export, click Clear.
  4. Click Create File. The export file is created in the IntegrationServer_Directory\packages\WmOAAdapter\exchange\export directory.
  5. When you are ready to import the exported transaction definitions, move the export file that is in the \export directory to the Integration Server_Directory\packages\WmOAAdapter\\exchange directory.
