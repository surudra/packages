BasicNotification Template Screens

The BasicNotification adapter notification polls a database table for data using a SELECT operation.

This adapter notification template includes the following tabs:

Tables Tab

Joins Tab

SELECT Tab

WHERE Tab

Tables Tab

Use the Tables tab to define the table, as follows:

Field
Description

Table Alias

The table alias is automatically assigned when you select more than one table in the Table Name field. The default is t1.

Table Name

Select a table name. The default for the associated catalog name is current catalog. The default for the associated schema name is current schema.

Type

The table type automatically displays based on the table you select.

Joins Tab

Use the Joins tab to specify the columns for joining the tables you configured on the Tables tab, as follows:

  1. Select the Insert Row icon (or the Fill in all rows to the table icon ) to create new left and right columns.
  2. Select Left Column and select the first table’s joining column.
  3. Select the appropriate Operator.
  4. Select Right Column and select the next table’s joining column.
  5. Repeat until you have defined all the joins.

SELECT Tab

Use the SELECT tab to define the columns and fields to be selected.

 

Note: If you plan on using the exactly once notification feature, be sure that you include the Record ID Column (described below) as one of the columns to be selected.

  1. In the ALL/DISTINCT field, select ALL to include duplicate rows or DISTINCT to suppress duplicate rows. Otherwise leave this field blank.
  2. Select the Insert Row icon (or the Fill in all rows to the table icon ) to create new fields as needed.
  3. In the Expression field, select a column or type any valid SQL expression. The corresponding Column Type, JDBC Type, Output Field Type, and Output Field display for each column you select in the Expression field. Use the following fields:

    4. Field
    Description

    Expression

    The column name or SQL expression.

    Column Type

    The column data type defined in the database table.

    JDBC Type

    The JDBC type of the corresponding Output Field.

    Output Field Type

    The data type of the output field. The JDBC Adapter automatically converts database-specific types to Java data types.

    For a list of JDBC-Type to Java-Type mappings, see the webMethods JDBC Adapter User’s Guide.

    Output Field

    The name of the field containing the output from the SELECT operation. An output field name displays when you select an expression.

    You can also modify the output field names as required.

    Sort Order

    Specifies ordering of publishable documents per each polling. Use this field to ensure that the notification’s publishable documents, for each polling, are in the correct ascending or descending order based on one or more table columns.

    Select either Ascend or Descend. Leave the field blank if there is no sort order.

  4. If you want the notification’s publishable document to be published to the local Integration Server, select the Publish Locally option.

    5. By default, this option is not selected, that is, if the webMethods Broker is configured to the Integration Server, the publishable document is published to the Broker; otherwise the publishable document is published to the local Integration Server.

    Selecting the Publish Locally option reduces performance problems, if the Integration Server is connecting to a remotely located Broker that is in turn triggering a service on the local Integration Server.

  5. If you want to use the Exactly Once notification feature, you must enable the Exactly Once Notification option. For more information, see the webMethods JDBC Adapter User’s Guide.
  6. Set the Delete selected records flag to automatically delete the selected records from the buffer table (based on their Record ID Column value as entered in step 7) after the notification. Use this option to prevent publishing the same documents to the Integration Server each time polling occurs.

    7. Running a BasicNotification may generate a duplicate message error; the Integration Server will ignore the duplicate notification document. In this case, you should check the Delete selected records option and choose a column with sequentially unique values as the Record ID Column value in the step below.

  7. You must use the Record ID Column field to use the Exactly Once notification feature. Select the column from the buffer table that you want to use as the unique ID for the publishable document for this notification. For more information, see the webMethods JDBC Adapter User’s Guide.

    8. To ensure that all values will be unique, choose a table column in the Record ID Column field whose values are sequential numbers.

  8. Use the Mark ID Column field to mark the records as processed. Select the column that you created in the database table to hold the status of the record. Use this option when you do not want to set the Delete selected records flag, and also want to avoid the publishing of duplicate records. For information about creating the column that holds the status of the records and marking the records as published, see the webMethods JDBC Adapter User’s Guide.
  9. Use the Maximum Row field to specify the maximum number of records to retrieve from the database. This field is useful when you are working with a large number of records and you want to limit the number of documents sent each time the notification polls. The default value of 0(no limit) retrieves all records.
  10. Use the Query Time Out field to specify the query time out value of the BasicNotification you are configuring. This value is the amount of time the JDBC Adapter waits for the notification to execute before stopping the SQL operation.

    11. The default value is -1. Use the default value to have the notification use the value indicated on the watt.adapter.JDBC.QueryTimeout property as the time out.

    If you specify a value equal to 0, or if the watt.adapter.JDBC.QueryTimeout property is not set, the notification executes without a time out.

    If you specify a value greater than 0, the notification executes with the specified value as the time out.

     

    Note: -1 is the only permissible negative value for this field.

    For more information about the watt.adapter.JDBC.QueryTimeout property, see the webMethods JDBC Adapter User’s Guide.

WHERE Tab

Use the WHERE tab to specify the WHERE conditions on the SQL query statement for selecting information:

  1. Select the Insert Row icon to define new WHERE clause fields.
  2. Select a logical operator from the AND/OR field, an Operator, and separators (the left and right parentheses) as needed, and use the following fields:

    3. Field
    Description/Action

    AND/OR

    The logical operator.

    Column

    The name of the column you want to use in the WHERE clause.

    Operator

    The operator used with the Column and Input Field.

    Input Field*

    Type a fixed value in this field. Be sure that it is a valid value, or an exception will be generated at run time.

     

    Note: * For Oracle users, if you use a CHAR(n) data type and enter a value in the Input Field, the JDBC Adapter automatically sets the ORACLEFIXED_CHAR data type as the JDBC data type.

  3. If necessary, use the Shift Up or Shift Down icons to change the order of the WHERE clause to ensure the parameters are parsed in the correct order.
  4. Repeat this procedure until you have specified all the WHERE parameters.
