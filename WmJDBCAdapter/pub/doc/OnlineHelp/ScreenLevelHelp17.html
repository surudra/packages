SelectSQL Service Template Screens

The SelectSQL adapter service retrieves specified information from a database table.

This service template includes the following tabs:

Tables Tab

Joins Tab

SELECT Tab

WHERE Tab

Tables Tab

Use the Tables tab to configure the database table (or tables) the operation accesses, using the following fields:

Field
Description

Table Alias

The system automatically assigns the table alias when you select more than one table in the Table Name field. The default is t1.

Table Name

Select a table name. The default for the associated catalog name is current catalog. The default for the associated schema name is current schema.

Type

The system automatically displays the type based on the table you select.

Joins Tab

If you want to join tables, use the Joins tab to specify the columns for joining the tables you just configured. If you are not joining tables, skip this tab.

  1. Select the Insert Row icon (or the Fill in all rows to the table icon ) to create new left and right columns.
  2. Select Left Column and select the first table’s joining column.
  3. Select the appropriate Operator.
  4. Select Right Column and select the next table’s joining column.
  5. Repeat this procedure until you have defined all the joins.

SELECT Tab

Use the SELECT tab to define the columns and fields to be selected.

  1. In the ALL/DISTINCT field, select ALL to include duplicate rows or DISTINCT to suppress duplicate rows. Otherwise leave this field blank, which is the default value.
  2. Use the Insert Row icon (or the Fill in all rows to the table icon ) to create new rows as needed.As you insert additional rows, the corresponding Column Type, JDBC Type, Output Field Type, and Output Field display for each column you select in the Expression field. Use the following fields:

    3. Field
    Description

    Expression

    The column name in the database table.

    Column Type

    The column data type defined in the database table.

    JDBC Type

    The JDBC type of the corresponding Output Field.

    Output Field Type

    The data type of the output field. The JDBC Adapter automatically converts database-specific types to Java data types.

    Output Field

    The name of the field containing the output from the SELECT operation. An output field name displays when you select an expression.

    You can also modify the output field names as required.

    Sort Order

    Specifies how rows are returned as follows:

    Select either Ascend or Descend. Leave the field blank if there is no sort order.

    Maximum Row

    Use this field only to specify the maximum number of records to retrieve from the database. The default value of 0 (no limit) retrieves all records.

    Query Time Out

    Specify the query time out value in seconds. This value is the amount of time the JDBC Adapter waits for the service to execute before stopping the SQL operation.

    The default value is -1. Use the default value to have the service use the value indicated on the watt.adapter.JDBC.QueryTimeout property as the time out.

    If you specify a value equal to 0, or if the watt.adapter.JDBC.QueryTimeout property is not set, the service executes without a time out.

    If you specify a value greater than 0, the service executes with the specified value as the time out.

    Note: -1 is the only permissible negative value for this field.

    For more information about the watt.adapter.JDBC.QueryTimeout property, see the webMethods JDBC Adapter User’s Guide.

    Result Field

    Specify a name for the output field that must contain the total number of rows affected by the SQL statement.

    Do not use results as the value of the Result Field.

    Result Field Type

    The data type of the Result Field.

WHERE Tab

Use the WHERE tab to specify the conditions for selecting information.

  1. Select the Insert Row icon to define new WHERE clause fields.
  2. Select an AND/OR/NOT logical operator from the AND/OR field, an Operator, and separators (the left and right parentheses) as needed, and use the following fields.
    Field
    Description

    AND/OR

    The logical operator.

    Column

    The name of the column you want to use in the WHERE clause.

    Operator

    The operator used with the Column and Input Field.

    Input Field*

    The default value is ?, which acts as a placeholder for the variable so that you can set the input variable for that column at run time, or get input external to this adapter service.

    You can also type a fixed value in this field now or at run time. If you type a fixed value, be sure that it is valid, or an exception will be generated at run time.

    JDBC Type

    The JDBC type of the corresponding Input Field.

    Input Field Type

    The corresponding input field’s Java type.

    For a list of JDBC-Type to Java-Type mappings, see the webMethods JDBC Adapter User’s Guide.

    Input Field
    (second occurrence of this field)

    Type the name of the input field.

    If you use the default ? variable placeholder as the Input Field value in the WHERE clause, be sure to enter the corresponding Input Field and its JDBC Field Type in the same order as they appear on the top portion of the WHERE tab.

     

    Note: *For Oracle users, if you use a CHAR(n) data type and enter a value in the Input Field, the JDBC Adapter automatically sets the ORACLEFIXED_CHAR data type as the JDBC data type.

     

    Note: The WHERE clause does not support the java.sql.array data type.

  3. If necessary, use the Shift Up or Shift Down icons to change the order of the WHERE clause to ensure the parameters are parsed in the correct order.
  4. Repeat this procedure until you have specified all WHERE parameters.
