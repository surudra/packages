OrderedNotification Template Screens

An OrderedNotification publishes notification data for multiple Insert, Update, or Delete operations on multiple tables.

This adapter notification template includes the following tabs:

Notification Configure Tab

Source Tables Tab

SELECT Tab

WHEN Tab

Notification Configure Tab

Use the Notification Configure tab as follows:

Field
Description

Base Name

The base name used to generate the Resource Name created by the JDBC Adapter.

The base name determines the corresponding Resource Types and Resource Name.

Note: For OS/390 DB2V7.2, the Base Name you create below must be no more than 5 characters because triggers on OS/390 name cannot be more than 8 characters.

Publish Locally

Specifies whether to publish the notification’s publishable document to the local Integration Server.

By default, this option is not selected, that is, if the webMethods Broker is configured to the Integration Server, the publishable document is published to the Broker; otherwise the publishable document is published to the local Integration Server.

Selecting the Publish Locally option reduces performance problems, if the Integration Server is connecting to a remotely located Broker that is in turn triggering a service on the local Integration Server.

Resource Type

Types are buffer table, trigger, and sequence.

For information about resource types, see the webMethods JDBC Adapter User’s Guide.

Resource Name

To ensure uniqueness, the resource name combines the following elements. You do not edit this name.

  • Resource prefix (WMB, WMT, and WMS for buffer table, trigger, and sequence respectively)
  • The name you typed in the Base Name field
  • A suffix, based on a system timestamp

File Record Format

Optional field for DB2 for AS400 V4R5 only.

Database Name

The name of the database where the buffer tables will be created.

Optional field used by DB2 for OS/390 only.

Table Space Name

The table space where the buffer tables will be created.

Optional field used by DB2 for OS/390 only.

Source Tables Tab

Use the Source Tables tab as follows:

Field
Description/Action

Table Alias

The table alias is automatically assigned when you select more than one table in the Table Name field. The default is t1.

Table Name

Select a table name. The default for the associated catalog name is current catalog. The default for the associated schema name is current schema.

For AS400 DB2 V4R5 version using a jt400.jar file, the table name for the notification cannot exceed 10 characters; otherwise an exception will be generated when you try to enable the notification.

Type

The table type displays automatically based on the table you select.

Operation Type

Select INSERT, UPDATE, or DELETE operation.

Operation ID

Assign an ID to unique identify the given operation for the notification.

SELECT Tab

Use the SELECT tab to define the columns and fields to be selected using the following fields:

  1. In the ALL/DISTINCT field, select ALL to include duplicate rows or DISTINCT to suppress duplicate rows. Otherwise leave this field blank, which is the default value.
  2. Select the Insert Row icon (or the Fill in all rows to the table icon ) to create new fields as needed. For each Expression column you select, the corresponding Operation ID, Column Type, JDBC Type, Output Field Type, and Output Field display.

    3.  

    Note: You can use the BigDecimal data type with the JDBC Adapter. However, the webMethods Developer does not support the BigDecimal data type. This means that the Developer will not correctly display a BigDecimal data type result and you cannot enter a value of this data type. However, the JDBC Adapter will process the BigDecimal data type correctly.

    Use the following fields:

    Field
    Description/Action

    Expression

    The column name.

    Operation ID

    The corresponding operation ID for the expression.

    Column Type

    The column data type defined in the database table.

    JDBC Type

    The JDBC type of the corresponding Output Field.

    Output Field Type

    The data type of the output field. The JDBC Adapter automatically converts database-specific types to Java data types.

    For a list of JDBC-Type to Java-Type mappings, see the webMethods JDBC Adapter User’s Guide.

    Output Field

    The name of the field containing the output from the SELECT operation. An output field name displays when you select an expression.

    You can also modify the output field names as required.

    Notify On Update

    Enable this option to indicate which of the columns specified in the SELECT tab you want notification if updated. Select:

    • Yes: you want notification if this column of data has been updated. Yes is the default value
    • No: you do not want notification if this column of data has been updated

      • For example, you configure the following three output fields: MyName, MyNumber, and MyLocation. You want notification only if the MyLocation output field is updated. In this case, you would select Yes for the MyLocation output field, and select No for the MyName and MyNumber output fields.

    Output Value Type

    Used for Update operations only. Specifies which output value to retrieve from the database table. By default, the notification retrieves the new value from the database table. Select either of the following output value types:

    • Old: Retrieves the old value from the database table
    • New: Retrieves the new value from the database table

    To retrieve both the old and the new values, you need to create two rows and then select an Output Value Type as Old in one row and an Output Value Type as New in the other row. While doing so, ensure that the Output Field is unique for both the old and the new values.

    Note: If you created Ordered Notifications for update operation using the JDBC Adapter 6.0.3, the values of this field, that is, New and Old are not displayed in the list. The workaround is to click the icon Reload values from the adapter .

    Maximum Row

    Specifies the number of rows to be retrieved from the buffer table. This field is useful when you are working with a large number of records and you want to limit the number of documents sent each time the notification polls.

    Use a value of 0 to indicate no limit on the number of rows retrieved.

    Query Time Out

    Specify the query time out value in seconds. This value is the amount of time the JDBC Adapter waits for the notification to execute before stopping the SQL operation.

    The default value is -1. Use the default value to have the notification use the value indicated on the watt.adapter.JDBC.QueryTimeout property as the time out.

    If you specify a value equal to 0, or if the watt.adapter.JDBC.QueryTimeout property is not set, the notification executes without a time out.

    If you specify a value greater than 0, the notification executes with the specified value as the time out.

    Note: -1 is the only permissible negative value for this field.

    For more information about the watt.adapter.JDBC.QueryTimeout property, see the webMethods JDBC Adapter User’s Guide.

WHEN Tab

Use the WHEN tab to specify the conditions for selecting information using the following table.

 

Note: If you use Microsoft SQL Server or DB2 AS400 using a jt400.jar file, do not use the WHEN tab because this feature is not supported. An exception will be generated if you try to use this tab in these cases.

  1. Select the Insert Row icon to define new WHEN clause fields.
  2. Select the Column field and choose a column from the list. The Operation ID will display after you make your selection.
  3. Select a logical operator from the AND/OR field, an Operator, and separators (the left and right parentheses) as needed.
  4. Type a fixed value in the Value field now or at run time. Be sure that it is a valid value, or an exception will be generated at run time.
  5. If necessary, use the Shift Up or Shift Down icons to change the order of the WHEN clause to ensure the parameters are parsed in the correct order.
  6. Repeat until you have specified all WHEN parameters.

     

    Note: While enabling the Ordered Notification that monitors a Sybase database, if you encounter an error that indicates data definition commands are not allowed in transactions, then you need to run the following command in your Sybase database: sp_dboption database_name,"ddl in tran", true.
