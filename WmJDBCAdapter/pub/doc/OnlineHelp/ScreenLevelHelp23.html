InsertNotification Template Screens

The InsertNotification adapter notification publishes notification of INSERT operations on a database table to the webMethods Integration Platform.

 

Note: InsertNotifications are not supported using Teradata databases.

This adapter notification template includes the following tabs:

Notification Configure Tab

Tables Tab

Joins Tab

SELECT Tab

WHEN Tab

Notification Configure Tab

Use the Notification Configure tab as follows:

Field
Description

Base Name

The base name used to generate the Resource Name created by the JDBC Adapter.

The base name determines the corresponding Resource Types and Resource Name.

Note: For OS/390 DB2V7.2, the Base Name you create below must be no more than 5 characters because triggers on OS/390 name cannot be more than 8 characters.

Publish Locally

Specifies whether to publish the notification’s publishable document to the local Integration Server.

By default, this option is not selected, that is, if the webMethods Broker is configured to the Integration Server, the publishable document is published to the Broker; otherwise the publishable document is published to the local Integration Server.

Selecting the Publish Locally option reduces performance problems, if the Integration Server is connecting to a remotely located Broker that is in turn triggering a service on the local Integration Server.

Resource Type

Types are buffer table, trigger, and sequence.

For information about resource types, see the webMethods JDBC Adapter User’s Guide.

Resource Name

To ensure uniqueness, the resource name combines the following elements. You do not edit this name.

  • Resource prefix (WMB, WMT, and WMS for buffer table, trigger, and sequence respectively)
  • The name you typed in the Base Name field
  • A suffix, based on a system timestamp

File Record Format

The format of the file record.

Optional field used by DB2 for AS/400 V4R5 only.

Database Name

The name of the database where the buffer tables will be created.

Optional field used by DB2 for OS/390 only.

Table Space Name

The table space where the buffer tables will be created.

Optional field used by DB2 for OS/390 only.

Tables Tab

Use the Tables tab to set the following fields:

Field
Description

Table Alias

The table alias is automatically assigned when you select more than one table in the Table Name field. The default is t1.

Table Name

Select a table name. The default for the associated catalog name is current catalog. The default for the associated schema name is current schema.

Type

The table type automatically displays based on the table you select.

Joins Tab

Use the Joins tab to specify the columns for joining the tables you configured on the Tables tab. If you are not joining tables, skip this tab.

  1. Select the Insert Row icon to create new left and right columns.
  2. Select Left Column and select the first table’s joining column.
  3. Select the appropriate Operator.
  4. Select Right Column and select the next table’s joining column.
  5. Repeat until you have defined all the joins.

SELECT Tab

 

Note: When using the Join clause, select only the fields of the monitor table in the Select tab.

Use the SELECT tab to define the columns and fields to be selected.

  1. In the ALL/DISTINCT field, select ALL to include duplicate rows or DISTINCT to suppress duplicate rows. Otherwise leave this field blank, which is the default value.
  2. Select the Insert Row icon (or the Fill in all rows to the table icon ) to create new fields as needed.
  3. In the Expression field, select a column or type any valid SQL expression. The corresponding Column Type, JDBC Type, Output Field Type, and Output Field display for each column you select in the Expression field. Use the following fields:

    4. Field
    Description

    Expression

    The column name or SQL expression.

    Column Type

    The column data type defined in the database table.

    JDBC Type

    The JDBC type of the corresponding Output Field.

    Output Field Type

    The data type of the output field. The JDBC Adapter automatically converts database-specific types to Java data types.

    For a list of JDBC-Type to Java-Type mappings, see the webMethods JDBC Adapter User’s Guide.

    Output Field

    The name of the field containing the output from the SELECT operation. An output field name displays when you select an expression.

    You can also modify the output field names as required.

    Maximum Row

    Specifies the number of rows to be retrieved from the buffer table. This field is useful when you are working with a large number of records and you want to limit the number of documents sent each time the notification polls.

    Use a value of 0 to indicate no limit on the number of rows retrieved.

    Query Time Out

    Specify the query time out value in seconds. This value is the amount of time the JDBC Adapter waits for the notification to execute before stopping the SQL operation.

    The default value is -1. Use the default value to have the notification use the value indicated on the watt.adapter.JDBC.QueryTimeout property as the time out.

    If you specify a value equal to 0, or if the watt.adapter.JDBC.QueryTimeout property is not set, the notification executes without a time out.

    If you specify a value greater than 0, the notification executes with the specified value as the time out.

    Note: -1 is the only permissible negative value for this field.

    For more information about the watt.adapter.JDBC.QueryTimeout property, see the webMethods JDBC Adapter User’s Guide.

WHEN Tab

Use the WHEN tab to specify the conditions for selecting information.

 

Note: If you use Microsoft SQL Server or AS/400 DB2 using a jt400.jar file, do not use the WHEN tab because this feature is not supported. An exception will be generated if you try to use this tab in these cases.

  1. Select the Insert Row icon (or the Fill in all rows to the table icon ) to define new WHEN clause fields.
  2. Select the Column field and choose a column from the list.
  3. Select an AND/ OR/NOT logical operator from the AND/OR field, an Operator, and separators (the left and right parentheses) as needed.
  4. Type a fixed value in the Value field now or at run time. Be sure that it is a valid value, or an exception will be generated at run time.
  5. If necessary, use the Shift Up or Shift Down icons to change the order of the WHEN clause to ensure the parameters are parsed in the correct order.
  6. Repeat until you have specified all WHEN parameters.
